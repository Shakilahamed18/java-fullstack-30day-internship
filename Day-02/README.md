# Day 02 – HTML5

## Learning Objectives

- Understand HTML5 semantic elements and their purpose
- Build accessible, well-structured web pages
- Create forms with proper input types and validation
- Apply accessibility best practices (ARIA, labels, alt text)
- Understand the document structure and metadata

## Topics Covered

- Document structure: `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`
- Semantic elements: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`
- Headings hierarchy (`<h1>` – `<h6>`)
- Forms: `<form>`, `<input>`, `<select>`, `<textarea>`, `<button>`
- Input types: text, email, password, number, date, checkbox, radio
- Form validation attributes: `required`, `minlength`, `pattern`
- Accessibility: `<label>`, `aria-*` attributes, `alt` text, semantic meaning
- Meta tags, `<link>`, and `<script>` placement

## Hands-On Task

Build a multi-section webpage for a fictional company:
1. Navigation bar with links
2. Hero section with heading and description
3. Services section using `<article>` elements
4. Contact form with: name, email, phone, message, and a submit button
5. Footer with copyright and social links

Ensure all form inputs have proper labels and the page passes basic accessibility checks.

## Practice / Homework

Run your page through the [W3C HTML Validator](https://validator.w3.org/) and fix any errors. Then test it with a screen reader or browser accessibility tools and note any improvements needed.
